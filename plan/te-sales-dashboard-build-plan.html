ระบบ Security — 5 ชั้นป้องกัน

ข้อมูลลูกค้า/มูลค่างานเป็นความลับทางธุรกิจ — ออกแบบสิทธิ์ตั้งแต่วันแรก ไม่ใช่มาเสริมทีหลัง

1️⃣ Authentication — ต้อง login ก่อนเห็นทุกอย่าง

Supabase Auth แบบ email + password · ปิดรับสมัครสาธารณะ — admin (เก๋) เป็นคนสร้าง/เชิญบัญชีให้ทีมเท่านั้น คนนอกเดา URL เจอก็เข้าไม่ได้ เห็นแค่หน้า login

2️⃣ Authorization — Row Level Security (RLS)

กติกาสิทธิ์บังคับที่ ตัวฐานข้อมูล ไม่ใช่แค่หน้าจอ: admin เห็น/แก้ทั้งหมด · sale เห็นงานทีมตัวเอง แก้ได้เฉพาะงานที่ตัวเองดูแล — ต่อให้มีคนพยายามยิง API ตรงก็ทะลุไม่ได้

3️⃣ การจัดการ Key

หน้าเว็บใช้เฉพาะ anon key (ออกแบบมาให้เปิดเผยได้ ความปลอดภัยจริงอยู่ที่ RLS) · service_role key ห้ามอยู่ใน repo เด็ดขาด · repo เป็น public (GitHub Pages ฟรี) จึงมีกติกา: ไม่มีข้อมูลลูกค้าหรือ secret ใดๆ ใน code — ข้อมูลทั้งหมดอยู่ใน DB เท่านั้น

4️⃣ การเชื่อมต่อ

HTTPS ทุกเส้นทางอัตโนมัติ (GitHub Pages และ Supabase บังคับอยู่แล้ว) — ใช้บน 4G/WiFi สาธารณะได้ปลอดภัย

5️⃣ Audit + Backup

ทุก record เก็บ created_by / updated_at อัตโนมัติ (รู้ว่าใครแก้อะไรเมื่อไร) · Supabase backup รายวันอัตโนมัติ + คงปุ่ม export JSON/CSV จาก prototype เดิมไว้เป็น backup มือ ก่อนปิดงานสำคัญกดเก็บได้เสมอ

✳️ ถ้าเลือกเส้นทาง VPS (SQLite/PocketBase)

ต้องเพิ่มเอง: JWT auth, HTTPS certificate (Let's Encrypt), rate limit, backup cron — คือเหตุผลหลักที่ Supabase เป็นตัวเลือกแนะนำสำหรับทีมที่ไม่มี dev ประจำ

  • Desktop (>1024px): เต็มจอเหมือน prototype เดิม

📲 ติดตั้งเป็น App (PWA)

iPhone/iPad: Safari → Share → Add to Home Screen · S24: Chrome → Install app — ได้ไอคอน TE บนหน้าจอ เปิดเต็มจอไม่มีแถบ browser เหมือน app จริง โดยไม่ต้องผ่าน App Store

⚡ เร็วบน 4G

Service worker cache ตัวหน้าเว็บไว้ในเครื่อง — เปิดครั้งถัดไปโหลดทันที ดึงเฉพาะข้อมูลใหม่จาก DB · สัญญาณหลุดยังเปิดดูข้อมูลล่าสุดที่ cache ไว้ได้ (การบันทึกต้อง online)
